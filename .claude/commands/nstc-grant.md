# 國科會（NSTC）專題研究計畫申請書撰寫 Skill

## Overview

本 Skill 協助大學教授撰寫符合**國家科學及技術委員會（NSTC）**標準的專題研究計畫申請書。
基於真實已通過計畫書格式設計，確保格式合規、論述架構完整、審查標準對齊。

**核心原則：計畫書是說服文件，必須同時展現研究創新性、重要性、可行性、社會影響力。**
每個章節需回答審查委員心中的問題：「這個研究為什麼重要？為什麼現在做？這個團隊能做到嗎？」

---

## 支援的計畫類型

| 計畫類型 | 適用對象 | 年限 | 特點 |
|---------|---------|------|------|
| **一般型研究計畫** | 所有教師 | 1~3年 | 最常見，本 Skill 主要支援 |
| **新進人員研究計畫** | 新進助理教授 | 1~2年 | 審查較寬鬆，重點在可行性 |
| **產學合作計畫** | 有業界合作者 | 1~3年 | 需附合作企業文件 |
| **優秀年輕學者** | 42歲以下 | 5年 | 需詳細研究願景 |

## CM03 核心章節架構（30頁上限）

### 一、研究計畫之背景（建議 8~12頁）

#### 1-1 研究背景與動機（3~5頁）

**寫作策略：**
- 開門見山點出當前技術/教育/社會現象的重大轉變
- 引用近期高影響力研究（2022年後優先）說明問題的急迫性
- 描述「現有研究/工具/方法的三大缺口」（具體、可量化）
- 結尾明確指向本研究的解決方案切入點

**必要結構：**
```
當前情境（現象描述）
    ↓
現有解法的限制（What's missing）
    ↓
三大研究缺口（編號列點，清晰）
    ↓
本研究定位（如何填補缺口）
```

**風格規範：**
- 中英術語並列：生成式人工智慧（Generative AI, GenAI）
- 每段最多5個bullet points
- 段落結束前加入「本研究將...」連接句，保持敘事流暢

#### 1-2 文獻探討（3~5頁）

**架構：按主題分節，非逐篇摘要**

建議分 3~5 個主題節：
- 每個主題 1~2 頁
- 每個主題結尾點出「現有研究的不足」→ 引導到本研究創新點

**引用規範：**
- APA格式（作者, 年份）：`（Kasneci et al., 2023）`
- 每頁引用密度：至少 3~5 筆
- 近3年文獻占比 ≥ 60%（高影響力期刊：SSCI/SCI/EI優先）
- 含台灣在地研究（顯示熟悉本土脈絡）

#### 1-3 研究目的（1~2頁）

**格式：**
```
一段敘述（50~100字）說明研究核心立場，
接著條列研究目的：

1. [動詞] + [研究對象] + [操作方式]：...
2. [動詞] + [研究對象] + [操作方式]：...
3. [動詞] + [研究對象] + [操作方式]：...
```

**動詞建議：建置、驗證、分析、發展、建構、探討、評估**

---

### 二、研究方法、進行步驟及執行進度（建議 10~15頁）

#### 2-1 研究方法（6~10頁）

**必要子節：**
1. **(一) 研究設計概述** — 整體研究架構圖（含各階段說明）
2. **(二) 研究場域與資料來源** — 樣本來源、資料蒐集方式、倫理說明
3. **(三) 研究工具/系統建置** — 具體技術規格、模組架構
4. **(四) 資料分析策略** — 分析方法對應研究目的
5. **(五) 預計可能遭遇之困難及解決途徑** — 至少列2~3個風險及應對

**技術研究寫法（AI/ML類）：**
```
平台架構描述 → 核心演算法/模型說明 → 資料處理流程 
→ 評估指標（量化） → 驗證策略
```

**教育研究寫法（資訊教育類）：**
```
研究設計（準實驗/問卷/觀察/行為資料）→ 研究場域與對象 
→ 測量工具（問卷量表/平台資料）→ 分析方法（SEM/HLM/NLP）
→ 倫理審查說明（IRB）
```

> ⚠️ **IRB 必要提醒**：凡涉及人體/學生的行為研究，**必須**在此節說明已送人文倫理審查（IRB），並於CM01勾選確認。否則計畫會被退件。

#### 2-2 預計遭遇困難與解決途徑（1~2頁）

**格式：逐項說明**
```
困難一：[描述技術/執行困難]
解決途徑：[具體的備案或方法]

困難二：...
```

---

### 三、預期成果（建議 4~6頁）

#### 3-1 預期完成之工作項目（2~3頁）

**必要：研究時程甘特表**（按月或季）

| 階段 | 期間 | 主要工作內容 | 預期成果 |
|------|------|------------|---------|
| 第一階段：... | 115年8月–11月 | - 工作項目1<br>- 工作項目2 | ‣ 成果1<br>‣ 成果2 |
| 第二階段：... | 115年12月–116年3月 | ... | ... |
| 第三階段：... | 116年4月–116年7月 | ... | ... |

**預期主要成果表：**

| 類別 | 預期成果 | 預期時間 |
|------|---------|---------|
| 技術研發 | [系統/平台/工具名稱] | 第X個月 |
| 研究數據 | [資料庫/指標/模型] | 第X個月 |
| 學術產出 | - 研討會論文 X~Y 篇（國內/國際）<br>- SSCI/Scopus 期刊論文 X 篇初稿 | 第X個月 |
| 技術報告 | [白皮書/技術文件] | 第X個月 |

> **學術產出基本要求（一般型1年期）**：至少 2~3 篇研討會論文 + 1 篇期刊論文初稿

#### 3-2 對於參與工作人員預期可獲之訓練（1頁）

說明研究助理（大學生/研究生）將獲得：
- 研究方法訓練
- 技術能力培訓
- 學術寫作與論文發表訓練

#### 3-3 預期效益（2頁）

**必要涵蓋四個面向：**
1. **學術貢獻** — 填補哪些研究缺口、提出哪些新知
2. **教育應用** — 對教師/學生/課程有何實質幫助
3. **技術應用與產學潛力** — 可否商業化、延伸應用
4. **社會與國家發展貢獻** — 對教育政策/數位轉型/人才培育的貢獻

---

### 四、參考文獻

**格式：APA 第7版**
```
作者姓, 名縮寫. (年份). 文章標題. 期刊名, 卷(期), 頁碼. https://doi.org/...
```

**數量建議：**
- 1年期計畫：20~40 筆
- 2~3年期計畫：40~60 筆

**來源組合建議：**
- SSCI/SCI 期刊：60%以上
- 國際研討會（IEEE/ACM/CHI）：20~30%
- 台灣在地研究（TSSCI/國科會報告）：10~20%

## 審查委員關注點（評分面向）

國科會審查採委員評分制，通常關注：

| 評分面向 | 說明 | 對應計畫書章節 |
|---------|------|--------------|
| **研究主題之創新性** | 是否提出新的問題框架、方法、或整合 | 1-1缺口說明 + 2-1研究設計 |
| **研究主題之重要性** | 對學術/社會/政策的實質貢獻 | 1-3研究目的 + 3-3預期效益 |
| **研究方法之可行性** | 時程是否合理、資源是否足夠 | 2-1方法 + 3-1甘特圖 |
| **主持人研究能力** | 相關發表紀錄、初步成果 | CM06主持人欄 + 文獻中自引 |
| **預期影響力** | 成果能否被引用、應用、延伸 | 3-3預期效益 |

## 領域特定寫作指南

### AI × 資訊教育（主力領域）

**關鍵詞組合（增加學門匹配度）：**
- 生成式人工智慧（GenAI）× 自我調控學習（SRL）
- AI素養（AI Literacy）× 學習分析（Learning Analytics）
- 教育巨量資料（Educational Big Data）× 行為資料分析
- 提示工程（Prompt Engineering）× 教學設計

**必引經典文獻：**
- Kasneci et al. (2023) - ChatGPT for good? Learning and Individual Differences
- Long & Magerko (2020) - What is AI literacy? CHI 2020
- Zimmerman (2000) - Self-efficacy and self-regulated learning
- Siemens & Baker (2013) - Learning analytics and EDM

**數據展示偏好：**
- SEM路徑模型（驗證性因素分析）
- 行為序列分析（Process Mining / Markov Chain）
- NLP語意分析（BERT embedding similarity）
- 群集分析（K-means, hierarchical）

---

### 機器學習 × 金融科技（次要領域）

**關鍵詞：**
- 資產定價（Asset Pricing）× 機器學習
- 財務工程（Financial Engineering）× 深度學習
- 股票預測（Stock Prediction）× LSTM/Transformer
- 投資組合優化（Portfolio Optimization）× Reinforcement Learning

**審查學門：** EC（財務金融）或 CS（電腦科學）

**數據展示偏好：**
- 實證資產定價（Empirical Asset Pricing）
- 夏普比率、最大回撤等財務指標
- 統計顯著性（p-value < 0.05）+ 效果量
- 與基準模型（CAPM/Fama-French）比較

## 完整計畫書撰寫流程

### Stage 1：前置準備（老闆提供資訊後，由 research-director 執行）

1. 確認計畫類型（一般型/新進/產學）
2. 確認研究學門代碼
3. 確認執行期限（幾年期）
4. 確認合作機構（如有）
5. 搜尋近3年相關文獻（由 literature-scout 執行）
6. 確認初步預算規模（總金額、助理人數）

### Stage 2：框架建立（grant-writer 執行）

1. 先寫「研究目的」3~4條（確認研究核心）
2. 建立研究設計概念圖
3. 規劃四大階段時程（含甘特表）
4. 估算初步預算

### Stage 3：章節撰寫（grant-writer 主導，literature-scout 提供文獻）

依此順序撰寫：
1. **1-1 研究背景與動機**（先寫，建立整體論述基礎）
2. **1-2 文獻探討**（literature-scout 提供文獻清單後展開）
3. **1-3 研究目的**（已有初稿，精修）
4. **2-1 研究方法**（最核心，需最多篇幅）
5. **3-1 執行進度**（甘特圖）
6. **3-3 預期效益**
7. **參考文獻**（citation-management 驗證格式）
8. **中英文摘要**（最後寫，提煉全文精華）

### Stage 4：經費規劃（grant-writer 執行）

1. 確認助理人數與月薪
2. 計算AI工具費（月費×12）
3. 列出設備清單（單價≥1萬元）
4. 估算國際會議差旅（1~2次）
5. 填入CM07/CM08/CM10表格

### Stage 5：審查與修訂（manuscript-reviewer 執行）

1. 核對每章節是否對應審查標準
2. 確認IRB說明是否完整
3. 驗證預算邏輯（人力費+耗材費+設備費≤合理上限）
4. 確認引用格式（APA第7版）
5. 中英文計畫名稱的學術性檢查

## 常見錯誤 × 注意事項

### ❌ 絕對不能犯的錯誤

1. **CM03 超過30頁** → 直接退件，不接受說明
2. **未勾選IRB審查** → 涉及學生的行為研究**必須勾選**
3. **研究目的與方法脫節** → 審查委員最容易抓的問題
4. **預算金額無法對應研究內容** → 每一筆費用都需說明研究用途
5. **引用文獻格式不統一** → 全篇必須統一使用APA或其他一種格式
6. **英文計畫名稱有文法錯誤** → 第一印象分直接扣分

### ⚠️ 高頻被退件原因

- 研究創新性不明確（和現有研究差異在哪？）
- 研究方法太模糊（如只說「使用機器學習」而不說什麼模型、什麼資料）
- 時程不合理（1年期計畫卻要跑3個大型實驗）
- 效益描述太空洞（說「具有重大社會貢獻」但沒有具體說明是什麼）
